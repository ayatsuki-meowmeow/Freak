## 使用する技術スタック

### FE/BE共通

- Typescript
  - 実務でもよく使っているもので慣れているため
- Zod
  - 実務でよく使っている。一通りは使えるが、より深く理解したい。また、FE/BE共通でバリデーションスキーマとして使えるため
- Bun
  - パッケージマネージャーとして高速とのことなので、使ってみたい。
  - API側はランタイムとしても使用する
- OpenAPI（スキーマファースト）
  - 実務でもスキーマ定義として使用経験あり。この定義を唯一の正とする。
  - FE/BEともにTS採用なのでより簡易な手段もあったが、将来的な技術スタックの移行への対応力を考慮するとこちらが良いという判断
  - スキーマを先に定義することでFE/BEのパラレル開発が可能になる
  - BEをGoなど他言語にリプレイスする際の技術的障壁を減らせる
  - コード生成には `openapi-typescript` を使用し、FEは `openapi-fetch`、BEはHonoのルーティングに型を適用する
- Turborepo
  - モノレポ管理をする上で、Vercelが開発のためNxと比較してNext.jsとの親和性が高い
  - ベンダーロックは好ましくないかもしれないが、バグ等に悩まされるリスクを下げて、開発に集中するため
